## Change Management

Substantial changes must identify:

1. **Motivation** — why this change is needed.
2. **Scope** — what is and is not changing.
3. **Compatibility impact** — breaking changes must be flagged.
4. **Validation plan** — how the change will be tested.
5. **Rollback path** — how to revert if something goes wrong.

## Release Process

Updates to the profile are applied by merging to `main`.
