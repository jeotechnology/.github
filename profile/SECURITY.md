# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| Latest `main` | Yes |
| Any prior release | No |

## Reporting a Vulnerability

> [!IMPORTANT]
> Do not open public GitHub issues for suspected security vulnerabilities.

Report security concerns privately:

```text
joshua.olds94@gmail.com
```

Include as much of the following as is safe to share:

- A clear description of the issue.
- Steps to reproduce, if safe to share.
- Affected versions, deployments, or configurations.
- Any known or suspected impact or exposure.

## Response

Security reports will be reviewed before any public disclosure. Do not publish exploit details, proof-of-concept code, or mitigation bypasses until a fix or mitigation has been coordinated.

## Scope

This policy covers the JEO Technology Organization Profile and its repository-managed source, configuration, workflows, and documentation.
